# Claude 使用状況ウィジェット ドラッグ移動対応 設計書

- **日付**: 2026-05-28
- **ステータス**: ドラフト（ユーザレビュー待ち）
- **対象ファイル**: `~/Library/Application Support/Übersicht/widgets/claude-usage.jsx`（このファイルのみ変更）
- **不変**: `~/.claude-usage-widget/fetch-usage.ts`, `usage.json`（データ取得系は一切触らない）

## 1. 目的

現在「画面右上に固定」されている Claude 使用状況ウィジェットを、**マウスのドラッグで自由に移動でき、位置を記憶する**（再起動後も維持）ように変更する。

## 2. 確定した前提（ブレインストーミングでの決定）

| 論点 | 決定 |
|------|------|
| 移動の意味 | ドラッグで自由移動＋位置記憶（四隅切替や固定変更ではない） |
| 実装方式 | 案A: ウィジェット内に自前ドラッグハンドラ＋`localStorage`（外部依存なし・1ファイル完結） |
| インタラクション前提 | Übersicht のインタラクションモード（ショートカット＋アクセシビリティ許可）を有効化する運用で合意 |
| ドラッグハンドル | ヘッダ行（"CLAUDE 使用状況"） |
| リセット操作 | ヘッダのダブルクリックで初期位置（右上）へ |

### Übersicht 仕様上の制約（調査結果）

- ウィジェットは既定で**クリックスルー**（マウス操作は背後デスクトップに抜ける）。
- クリック/ドラッグを受け取るには **①インタラクション用ショートカット設定 ②アクセシビリティ許可** が必要。**ショートカットでインタラクションモードをONにしている間だけ**ウィジェットがマウス操作を受け取れる。
- 出典: https://github.com/felixhageloh/uebersicht#readme
- `className` は静的な文字列エクスポートのため、**動的な座標はインラインstyleで指定**する必要がある。
- 全ウィジェットは単一 webview 上に描画され、**`localStorage` は Übersicht 再起動を跨いで永続**する。
- `render` は `refreshFrequency`（本ウィジェットは30秒）ごとに**関数として再実行**される。React hooks の状態永続は不安定なため、ドラッグ状態は **React stateに依存しない**設計とする。

## 3. アーキテクチャ / 振る舞い

### 3.1 位置の保持と永続化

- 位置を `{x, y}`（px・画面左上原点）で表現。
- 永続キー: `localStorage["claude-usage-pos"]`（JSON）。
- 読み込み順:
  1. `localStorage` に保存があればそれを使用。
  2. 無ければ初期値（現状と同じ右上）= `x = window.innerWidth - PANEL_WIDTH - MARGIN`, `y = MARGIN`（`PANEL_WIDTH=320`, `MARGIN=14`）。
- 位置はモジュールスコープ変数 `pos` にも保持し、30秒再描画で失われないようにする（`localStorage` が正・`pos` はキャッシュ）。

### 3.2 配置方法

- 現状 `className` の `top:14px; right:14px;` 固定をやめる。
- `className` のラッパは原点 `top:0; left:0;` に置き、**実パネルは `render` 内インラインstyleで `position: fixed; left:${pos.x}px; top:${pos.y}px;`** として配置。
- パネル root に安定した `id="claude-usage-panel"` を付与（ドラッグ時のDOM取得用）。

### 3.3 ドラッグ（React stateに依存しない方式）

- ヘッダ行に `onMouseDown` と `cursor: move` を付与（ドラッグハンドル）。
- `onMouseDown(e)`:
  - ドラッグ開始位置とパネル現在座標の差分（offset）を記録。
  - `window` に `mousemove` / `mouseup` リスナを**その場で登録**（多重登録防止のため `mouseup` で必ず解除）。
  - テキスト選択抑止（`e.preventDefault()` / `userSelect:none`）。
- `mousemove(e)`:
  - 新座標 = カーソル位置 − offset。`clamp()` で画面内に補正。
  - パネル要素の `style.left/top` を**直接更新**（描画の即応性）＋ モジュール変数 `pos` を更新。
- `mouseup`:
  - `pos` を `localStorage` に保存。`mousemove`/`mouseup` リスナを解除。

### 3.4 画面外防止（クランプ・純粋関数）

```
clampPos(x, y, panelW, panelH, screenW, screenH) -> {x, y}
  x: 0 .. (screenW - panelW)
  y: 0 .. (screenH - panelH)   // 最低でも全体が画面内に残る
```

- パネル高さ `panelH` は実DOM（`offsetHeight`）から取得。取得不可時はフォールバック定数。
- この関数はテスト容易性のため副作用なしの純粋関数として切り出す。

### 3.5 リセット

- ヘッダ `onDoubleClick`: `localStorage["claude-usage-pos"]` を削除し、`pos` を初期値（右上）に戻して再配置。

## 4. ユーザ手動セットアップ（一度きり・実装後に手順案内）

1. Übersicht 環境設定でインタラクション用ショートカットを割り当てる。
2. システム設定 > プライバシーとセキュリティ > アクセシビリティ で Übersicht を許可。
3. 運用: ショートカットでモードON → ヘッダをドラッグで移動 → モードOFF（位置は記憶）。

## 5. テスト / 検証方針

### 自動（単体テスト可能な純粋ロジック）
- `clampPos()`: 画面内/上下左右端/画面外入力に対し境界内へ補正されること。
- 初期位置計算: 与えた `innerWidth` に対し右上座標が正しいこと。
- `localStorage` シリアライズ/デシリアライズの往復一致。

### 手動（GUI・Übersicht上）
1. ウィジェットをリロード → 既定で右上に表示。
2. インタラクションモードON → ヘッダをドラッグ → 任意位置へ移動。
3. モードOFF → 位置が維持され、通常表示（クリックスルー）に戻る。
4. Übersicht 再起動 → 位置が復元される。
5. ヘッダをダブルクリック → 右上へ復帰。
6. 画面端へドラッグ → パネルが画面外に消えない。

## 6. スコープ外（YAGNI）

- 四隅スナップ / グリッド吸着。
- 複数ディスプレイ間の座標の自動移動（initialPos は表示中ディスプレイ基準。load時クランプで画面内には収める）。
- 表示項目の追加・コスト換算・リポジトリ化（別タスク）。

## 7. 実装後の調整（2026-05-28 GUI検証で判明・追加修正）

実機（内蔵Retina 2560×1664 ＋ 外部1920×1080 のマルチモニタ）で確認したところ、設計に3つの穴があり修正した。

1. **ロード時クランプの欠落 → 画面外で復帰不能**
   - 当初クランプはドラッグ中のみ。ドラッグ時と表示時でディスプレイ論理幅が異なると、保存位置が画面外に出てハンドルに触れず戻せなくなる事象が発生（実際に保存値 `{x:1114,y:6}` が現ディスプレイで右にはみ出した）。
   - 対処: `loadPos()` 末尾でも `clampPos` を適用。画面サイズ変更・別ディスプレイ・不正な保存値でも必ず画面内へ。

2. **端ぴったり（余白ゼロ）になる**
   - クランプ上限が `screenW - panelW`（余白0）だったため、復帰時にパネルが画面端へ張り付いた。
   - 対処: `clampPos` に `margin` 引数（既定0）を追加し、load/drag で `MARGIN` を渡して端から余白を保つ。

3. **ボックスモデルによる右端はみ出し（約12px）**
   - `shellStyle` の `width:320` は content-box のため `padding(24)+border(2)` が加算され実幅≈346px。位置計算の `PANEL_WIDTH(320)` と不一致で右端が画面外へはみ出していた。
   - 対処: `shellStyle` に `boxSizing:"border-box"` を追加し実幅を320pxに一致させた。

### 余白の最終仕様
- 上端マージン `MARGIN = 8`、右端マージン `MARGIN_RIGHT = 8`（独立した定数。非対称調整が可能）。
- `initialPos` は `x = screenW - PANEL_WIDTH - MARGIN_RIGHT`, `y = MARGIN`。
- 純粋ロジック（`clampPos` の margin 対応含む）は `lib/geometry.js` のテスト10件で担保。

### 残注意（既知の結合）
- `MARGIN_RIGHT < MARGIN` のように右を上より小さくしたい場合、load/drag のクランプ margin（現在 `MARGIN`）が右余白を押し戻すため、その際はクランプ margin を両者の最小値にする調整が必要。現状は両方 8 のため問題なし。
